# Named Framework Development

This skill produces a plan for developing a named framework, methodology, model, or concept that practitioners reference by name. The output is a multi-month workstream: identify the insight worth naming, name it well, publish the canonical explainer, distribute aggressively, and build the citation flywheel.

This is one of the highest-leverage AEO investments available. AI engines disproportionately cite named frameworks because they are unambiguous reference points. "The Inbound Marketing methodology" cites HubSpot. "Jobs to Be Done" cites Christensen. "The Flywheel" cites Amazon. "The Lean Startup" cites Ries. Every external reference to the framework is an implicit citation of the brand. The compounding effect builds a moat that competitor content campaigns struggle to match.

The investment is meaningful: typically 3-6 months from concept to publication, then 6-18 months of distribution before the framework starts compounding on its own. Most brands never attempt it because the timeline is long. The brands that do attempt it often produce the dominant entity in their category for a decade.

## When to Run This Skill

Run it when:

- The `ai-search-audit` Check #6 returned no named framework owned by the brand
- The brand has 5+ years of experience in a category and has accumulated genuine insight
- The brand has a content program that needs a center of gravity
- The competitor analysis shows that no competitor in the space owns a named framework (open field)
- The user is willing to commit 3-6 months to development and 12+ months to distribution

Skip it when:

- The brand is too new to have genuine insight worth naming (force-naming generic concepts produces forgettable frameworks)
- The category already has well-established frameworks owned by larger brands (carving out a competing concept is much harder than naming something fresh)
- The brand cannot commit to the multi-year distribution work (a named framework without distribution does not propagate)
- The user wants a quick AEO win (this is not it)

## Before You Start

Read `client-context.md` for the brand's voice, the category, and the customer's actual language. Named frameworks succeed when they use the customer's words, not the brand's internal jargon. A framework named in the agency's terminology will not get picked up by practitioners.

Also pull the existing list of frameworks in the category. Many practitioners can name 5-15 established frameworks in their field. Your candidate framework needs to be genuinely novel; you cannot rename a concept that already has a widely-used name.

## The Five-Stage Development Path

The path from "we should have a framework" to "people cite our framework" runs in this sequence.

### Stage 1: Find the Insight (Weeks 1-4)

The insight is the thing the framework points at. It must be:

- **Non-obvious.** If the insight is widely accepted, it cannot anchor a new framework. "Customer experience matters" is not a framework candidate.
- **Specific to the brand's domain.** The brand should have genuine expertise here. Frameworks invented by people who do not practice the underlying craft are detected and dismissed by practitioners.
- **Useful.** Practitioners must want to reference it. A framework no one needs to use does not propagate.
- **Defensible.** When challenged, the brand can point to specific evidence (data, case studies, years of practice) that supports the framework. Unsupported frameworks get debunked.

How to find it:

1. **Interview the brand's senior practitioners.** Owner, founder, senior consultants, anyone with 10+ years in the category. Ask: "What do you know about [category] that most practitioners get wrong?" "What pattern do you see across clients that no one talks about?" "If you had to teach a new hire one non-obvious thing, what would it be?"
2. **Mine the brand's case studies and project records.** Look for repeated patterns. The same customer mistake across 50 projects. The same fix that works across 20 industries. The same metric that predicts success.
3. **Audit the existing thought leadership in the category.** What is everyone already saying? The framework opportunity sits in the negative space: the topic everyone dances around but no one has named.
4. **Test the candidate insight.** Share it with 5-10 practitioners in the category (without naming it yet). Do they recognize the pattern? Do they care about it? If practitioners say "huh, never thought of it that way, but yes," the insight has traction. If they say "yeah, everyone knows that," it is not novel enough.

Output: a one-paragraph statement of the insight, validated by practitioner reactions.

### Stage 2: Name the Insight (Weeks 4-6)

A bad name kills a good framework. A good name carries a mediocre framework further than its substance deserves.

Criteria for a good framework name:

- **Memorable.** Easy to say and hear once. Survives the office-conversation test ("Yeah, we should use the [framework] for this.").
- **Descriptive but not literal.** Hints at what the framework does without being a generic phrase. "Jobs to Be Done" hints; "Customer Need Identification Framework" does not.
- **Has a citable identity.** "The Flywheel" is citable; "The Marketing Loop System" is not. Brevity wins.
- **Search-resilient.** Search the candidate name. Does it return clean results, or is it already buried in other meanings? "The Pyramid Principle" works; "The Pyramid Method" is already taken several times.
- **Voice-consistent.** The name should sound like something the brand would actually say. A roofer's framework should not sound like a SaaS framework.

Common patterns that work:

- **Noun + descriptor:** "The Flywheel," "The Pyramid Principle," "The North Star Metric"
- **Verb + noun:** "Jobs to Be Done," "Crossing the Chasm"
- **Acronym from a meaningful sequence:** "RACI," "SMART goals," "RICE"
- **Named after a structural metaphor:** "The Funnel," "The Cone," "The Iceberg"

Common patterns that fail:

- **Long phrases requiring memorization.** "The Six-Step Customer Acquisition Optimization Framework" is dead on arrival.
- **Brand name embedded in the framework name.** "The Cardinal Ridge Method" cannot propagate beyond Cardinal Ridge customers.
- **Generic words.** "The Approach," "The Method," "The System" disappear into noise.
- **Names that need explanation in the name.** "The Three-Pillar Conversion Architecture" tries to do too much in the title.

Process:

1. Brainstorm 20-30 candidate names. Most will be bad. The bad ones surface what is wrong with the framework as much as they surface naming options.
2. Cut to a short list of 5-8.
3. Test the short list with practitioners. "Which of these names would you remember a week from now?" "Which one would you actually say out loud to a colleague?"
4. Pick the one that survives both tests. Run a domain check (the framework name typically becomes a URL slug on the brand's site).
5. Lock the name. Resist renaming after launch; renames reset the propagation work.

### Stage 3: Publish the Canonical Explainer (Weeks 6-12)

The canonical explainer is the single page that introduces the framework, explains it definitively, and serves as the citation target whenever the framework is referenced.

Structure:

- **Top of page:** Name + one-sentence definition. The most quotable sentence on the site.
- **Why it exists:** the insight the framework addresses, in 2-3 paragraphs.
- **What it is:** the framework's structure (steps, components, principles), explained with diagrams or visual representation where possible.
- **How to apply it:** worked examples showing the framework in action. Specific case studies, real numbers, named scenarios.
- **Common misuses:** anti-patterns, edge cases, where the framework does not apply.
- **FAQ:** real questions practitioners will have. Match real query language.

Length: substantive. 2,000-5,000 words for the canonical page. Padding is detected; depth is rewarded.

Voice: the brand's voice, applied. Authoritative but not corporate. Specific examples beat abstract claims.

The canonical explainer is built using the patterns from `content-for-citations`: inverted pyramid, quotable sentences front-loaded, self-contained passages, question-style headings, factual specificity.

The page URL becomes the citation target: `[brand]/resources/[framework-slug]/`. Schema it as Article + correctly-linked Person author + Organization publisher.

### Stage 4: Build the Supporting Content Cluster (Weeks 10-20)

The canonical explainer is the hub. Around it, build:

- **Worked-example posts** showing the framework applied to specific industries, customer types, or scenarios.
- **Comparison posts** showing the framework alongside existing frameworks in the category. ("X framework vs Jobs to Be Done")
- **Diagnostic posts** helping practitioners identify when the framework applies and when it does not.
- **Tool resources** (templates, worksheets, calculators) that operationalize the framework.
- **Adjacent thought-leadership posts** introducing related concepts that orbit the framework.

8-15 supporting posts is a reasonable target. Each links to the canonical explainer with descriptive anchor text. The cluster becomes the topical authority signal.

### Stage 5: Distribute Aggressively (Months 4-18)

A named framework without distribution is just a page on a website. Distribution is the work that makes the framework a citation source.

Distribution tactics (use all):

- **Speak about it.** Podcast appearances, conference talks, webinars, panel discussions. Every appearance is an opportunity to introduce the framework by name to a new audience.
- **Write about it externally.** Guest posts in industry publications, bylined articles in trade press, contributions to round-up posts. Reference the framework by name; link back to the canonical explainer.
- **Pitch it as a story angle.** Reporters covering the category may use the framework as a structural device for their pieces if it is interesting and well-explained.
- **Teach it.** Workshops, training programs, courses. Teaching builds the strongest version of adoption.
- **Tool it up.** Templates, calculators, frameworks-as-products that operationalize the framework. The Lean Canvas operationalizes Lean Startup; the Inbound Marketing methodology was operationalized by HubSpot's product itself.
- **Get others to publish with it.** Practitioners who find the framework useful will write their own takes on it. This is the strongest signal of adoption. The brand can encourage this by making the framework freely usable, by being responsive to questions about it, and by amplifying others who use it.
- **Reference it in every brand communication.** Sales calls, client work, internal training, blog posts on adjacent topics. The brand using its own framework consistently is part of how others learn to use it.

The flywheel: a framework becomes self-sustaining when it has been cited enough times in third-party content that practitioners learn the name without ever reading the canonical explainer. Practitioners reference it; new practitioners search for it; the brand becomes the source they find.

Realistic timeline to flywheel:

- **Months 1-6 of distribution:** active push. Few external citations.
- **Months 6-12:** first occasional external citations from practitioners in the brand's network.
- **Months 12-18:** broader uptake as the framework propagates through the category.
- **Years 2-3:** the framework appears in third-party content the brand had no hand in producing. The flywheel is engaged.

## When the User Pushes Back

**"Just name something we already do."** Almost always a mistake. Practitioners detect retrofitted naming. The framework needs to point at a genuine insight, not just relabel existing work. If the brand cannot identify a genuine non-obvious insight, the brand is not yet ready for framework development. Build more practice and case-study volume first.

**"This will take too long."** Yes. It is the longest-payoff AEO investment available. If the brand needs a 90-day win, this is not it. The brands that compound on a named framework are the brands that wait the 18-24 months.

**"Should we trademark it?"** Usually no. Trademarking a framework typically backfires: practitioners avoid referencing the framework if they worry about legal exposure, which kills the propagation. The brand's leverage is in being the citation source, not the trademark holder. Frameworks that have been trademarked (e.g., MEDDIC) usually became popular before the trademark, not because of it.

**"What if a competitor copies the framework?"** Likely, eventually. The defense is being the primary source. Practitioners and AI engines look for the original; the canonical explainer with the strongest entity signals wins. Some of the most-cited frameworks (Jobs to Be Done, Lean Startup, RICE) have been copied, adapted, and re-explained thousands of times; the original authors still get the citation share.

## Common Pitfalls

- **Naming a generic concept.** "The Customer Centricity Framework" cannot win because customer centricity is already a generic phrase. The framework name has to be specific and ownable.
- **Embedding the brand name in the framework name.** "The Cardinal Ridge Roof Inspection Method" can never become a generic reference. The name should be brand-agnostic enough that others use it.
- **Publishing the canonical explainer and stopping.** Distribution is 80% of the work. Without it, the framework sits unread.
- **Treating it as one writer's project.** Named frameworks require multi-stakeholder commitment: the senior practitioner who owns the insight, the writer who shapes the explainer, the marketing function that distributes, the leadership that uses it consistently.
- **Changing the framework once it has launched.** Iteration kills propagation. If the framework is wrong, kill it and start over with a new name; do not rebrand or restructure mid-flight.
- **Launching the framework before the brand has the authority signals to support it.** A framework is most credible when published by an established expert with strong entity authority. Brands without that authority should build it first; otherwise the framework reads as "random startup invented a thing nobody asked for."
